## Parte 2: Visualizer Rendering Refactor

### O problema

Cada visualizer plugin era responsável por:
1. Gerar a figura (plotly, matplotlib, D3)
2. Escolher formato (PNG, HTML, SVG)
3. Serializar pra arquivo (`fig.write_image()`, `plt.savefig()`)
4. Gerenciar I/O (output_path, criar diretórios)

Isso criava:
- **Duplicação**: cada plugin reimplementava lógica de formatos e I/O
- **Dependência frágil**: kaleido (100MB, Chromium headless) necessário pra PNG de plotly, quebra em containers/CI
- **Fricção pro plugin author**: precisa conhecer serialização, output_path, formatos
- **Acoplamento**: plugin decidia formato, mas quem deveria decidir é o consumer

Com cada plugin novo, essa complexidade se multiplicava.

### A solução

**Plugin retorna objeto de figura nativo. BaseClass serializa.**

```
ANTES:
  Plugin._render_impl(data, config, output_path)
    → gera figura
    → escolhe formato
    → serializa pra arquivo
    → retorna Path

DEPOIS:
  Plugin._render_impl(data, config)
    → gera figura
    → retorna plotly.Figure / matplotlib.Figure / HTML str

  BaseVisualizerPlugin._serialize(fig, output_format)
    → detecta tipo via duck-typing
    → serializa pro formato pedido pelo consumer
    → retorna {"html": str} ou {"data": base64, "encoding": "base64", "format": str}
```

### Como funciona o duck-typing

O `_serialize()` não importa plotly nem matplotlib. Ele detecta o tipo da figura por capacidade:

```python
def _serialize(self, fig, fmt):
    if isinstance(fig, str):          # HTML puro
        return {"html": fig}

    if hasattr(fig, 'to_html'):       # plotly.Figure
        if fmt == "html":
            return {"html": fig.to_html(include_plotlyjs="cdn", full_html=True)}
        elif fmt in ("png", "svg"):
            return {"data": base64(fig.to_image(format=fmt)), ...}

    if hasattr(fig, 'savefig'):       # matplotlib.Figure
        if fmt == "html":
            return {"html": self._matplotlib_to_html(fig)}  # PNG inline em <img>
        elif fmt in ("png", "svg"):
            return {"data": base64(fig.savefig(buf, format=fmt)), ...}
        finally:
            plt.close(fig)  # sempre fecha, mesmo em erro

    raise TypeError(f"Tipo não suportado: {type(fig)}")
```

Vantagens:
- Sem imports de plotly/matplotlib no BaseClass (lazy por design)
- Qualquer lib que tenha `to_html()` ou `savefig()` funciona automaticamente
- matplotlib Figure sempre é fechado via `try/finally` (sem memory leak)

### Detecção dinâmica de formatos

Cada plugin declara `RENDER_LIB` como atributo de classe:

```python
class FrequencyChartPlotly(BaseVisualizerPlugin):
    RENDER_LIB = "plotly"
```

O `get_supported_formats()` usa isso + detecção de libs instaladas:

```python
@staticmethod
def get_supported_formats(render_lib):
    if render_lib == "html":
        return ["html"]                        # sempre disponível
    elif render_lib == "matplotlib":
        return ["html", "png", "svg"]          # matplotlib gera tudo nativo
    elif render_lib == "plotly":
        formats = ["html"]
        if importlib.util.find_spec("kaleido"):
            formats.extend(["png", "svg"])     # kaleido instalado? PNG/SVG ok
        return formats
```

O plugin expõe isso no `meta().parameters.output_format.options`. O frontend `ParamForm` renderiza o dropdown automaticamente — mostra só formatos que funcionam. Se kaleido não está instalado, o dropdown mostra só "HTML". Sem surpresa pro usuário.

### Fluxo completo: API request → response

```
Consumer: POST /visualize/frequency_chart_plotly
  body: {data: {word_frequencies: {...}}, config: {chart_type: "bar"}, output_format: "html"}

Rota (visualize.py):
  1. require_plugin_type(core, "frequency_chart_plotly", "visualizer")  → ok
  2. validate_plugin_config(core, "frequency_chart_plotly", config)     → ok
  3. config = {chart_type: "bar", output_format: "html"}
  4. result = await to_thread(plugin.render, data, config)              → dict

BaseVisualizerPlugin.render():
  1. config_copy = dict(config)                   → não muta original
  2. output_format = config_copy.pop("output_format")  → "html"
  3. validated = _validate_config(config_copy)     → tipo convertido, defaults aplicados
  4. _validate_data(data)                          → word_frequencies presente? ok
  5. fig = _render_impl(data, validated)           → plotly.Figure
  6. return _serialize(fig, "html")                → {"html": "<html>...plotly...</html>"}

Rota retorna:
  {"status": "success", "plugin_id": "frequency_chart_plotly", "html": "<html>..."}

Frontend ResultView:
  detectType(r) → r.html existe → 'html'
  → <HtmlResult html={result.html} />
  → <iframe srcdoc="<html>..." sandbox="allow-scripts" />
```

### O que saiu do codebase

- `output_path` em toda a chain (interface → base class → engine → rota → plugins)
- `format` como parâmetro manual em cada plugin
- Temp file creation/cleanup na rota `/visualize`
- `FileResponse` + `BackgroundTask` cleanup
- Base64 encoding na rota (moveu pro BaseClass)
- matplotlib Agg backend setup no BaseClass
- `_ensure_path()` helper
- Diretórios antigos: `plugins/wordcloud_viz/`, `plugins/frequency_chart/`, `plugins/sentiment_viz/`

### O que entrou

- `RENDER_LIB` class attribute nos plugins
- `_serialize()` no BaseClass — duck-typing → serialização
- `_matplotlib_to_html()` — converte Figure pra `<img>` inline
- `get_supported_formats()` — detecção dinâmica de libs
- `_validate_config()` unificada (mesma lógica do BaseAnalyzerPlugin)

### Impacto no plugin author

**Antes — criar um visualizer novo:**
```python
class MeuPlugin(BaseVisualizerPlugin):
    def _render_impl(self, data, config, output_path):
        import plotly.graph_objects as go
        fig = go.Figure(go.Bar(x=[...], y=[...]))

        # Qual formato? Onde salvar? Como serializar?
        if config.get("format") == "html":
            fig.write_html(str(output_path))
        else:
            fig.write_image(str(output_path))  # precisa kaleido!

        return output_path
```

**Depois — criar um visualizer novo:**
```python
class MeuPlugin(BaseVisualizerPlugin):
    RENDER_LIB = "plotly"

    def _render_impl(self, data, config):
        import plotly.graph_objects as go
        fig = go.Figure(go.Bar(x=[...], y=[...]))
        return fig  # pronto
```

Uma responsabilidade: dados → figura. Sem I/O, sem formatos, sem kaleido.
